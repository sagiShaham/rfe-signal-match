Scoring Configuration

🧮 How Scoring Works

1. Similarity per source
Each context document is compared against the RFE text using semantic similarity (0.0–1.0 raw score).
2. Weight multiplier
Raw score × source weight = weighted signal. A Release Notes match at 0.5 similarity → 0.5 signal. A Confluence page at 0.5 → 0.15 signal.
3. Recency decay
Sources older than 12 months have their weight reduced by 20%. A 13-month-old doc is still relevant — just slightly discounted.
4. Strongest signal wins
The system takes the max signal across all sources — not an average. One strong Release Notes hit dominates over many weak Confluence matches.
Status vs Confidence are separate dimensions. Confidence (High/Medium/Low) reflects how strongly the sources match the RFE text. Status (Delivered/In PI/Planned/No Match) is determined by which source triggered the match — not how strong the match was.

📊 Source Weights

Higher weight = stronger signal. When multiple sources match, the highest-priority source wins: Release Notes → ADO Current PI → ADO Upcoming PI → Roadmap → Confluence.
